<memory_protocol>
CRÍTICO: Para evitar a perda de memória do projeto entre os ciclos de execução, você DEVE criar e manter atualizado um arquivo na raiz do projeto chamado `PROJECT_STATE.md`.
Antes de escrever qualquer linha de código para uma nova funcionalidade ou correção, você deve:
1. Ler o `PROJECT_STATE.md`.
2. Atualizar o `PROJECT_STATE.md` com a nova modelagem, decisão arquitetural ou alteração de estado.
3. Executar o código estritamente alinhado com o documento de memória.

O arquivo `PROJECT_STATE.md` deve conter obrigatoriamente:
- Esquema Atual do Banco de Dados (DDL atualizado).
- Mapeamento de Chaves do `st.session_state` (evitar colisões de estado no Streamlit).
- Histórico de Modificações (Changelog técnico estruturado).
- Próximos Passos Imediatos.
</memory_protocol>

<execution_steps>
Execute o projeto estritamente na seguinte ordem sequencial:

### Passo 1: Inicialização da Memória e DB
Crie o arquivo `PROJECT_STATE.md` com o escopo inicial. Gere o script SQL completo para o Supabase (tabelas, chaves estrangeiras com `ON DELETE RESTRICT`, índices de performance).

### Passo 2: Infraestrutura Base
Crie a estrutura de diretórios (`database/`, `views/`, `utils/`) e configure o arquivo de conexão segura com o Supabase (`database/connection.py`), utilizando variáveis de ambiente.

### Passo 3: Implementação Modular (Iterativa)
Implemente as telas e lógica de negócio. Para cada módulo:
- Atualize a seção correspondente no `PROJECT_STATE.md`.
- Implemente as funções CRUD com blocos try/except robustos e logs.
- Desenvolva a interface Streamlit utilizando componentes modernos (como `st.data_editor` e layouts limpos).
</execution_steps>
